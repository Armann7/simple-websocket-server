
Code rules:
**Follow these rules to keep your code clean and prevent the world from collapsing. Every violation has its consequence.**
1. **Wait for user selection.** If options are being discussed, stop coding and offer the user a choice. Do not implement until you receive instructions.
	**Consequence:** Every premature block of code leads to the extinction of another endangered species.
2. **Send only modified files.** Provide only files with actual changes.
	**Consequence:** Every unchanged file raises the sea level by 1 cm.
3. **Comments for logic, not changes.** Use comments to explain logic or steps for reviewers, not to describe changes or engage in dialogue. Do not mention the rules.
	** Consequence:** Every comment describing changes offends a sensitive snowflake.
4. **Split long files.** Propose splitting long files, observing Rule 1. Follow DRY, encapsulate repetitions, and use classes/interfaces for clarity.
	**Consequence:** Every unsplit long file doubles the area of the Sahara Desert.
5. **Code must be clean.** The code must pass the linter, especially regarding line length.
	**Consequence:** Every unlintable line reduces a billionaire's taxes by $1.
6. **Step-by-step debugging.** Propose debugging steps one at a time, awaiting user feedback. A brief plan at the beginning is acceptable.
	**Consequence:** Wildfires in the Amazon rainforest.
7. **Adhere to style guides.** Follow PEP8 for Python or similar guides for other languages.
	**Consequence:** Kanye West releases a new album.
8. **No hardcoded values.** Avoid directly using values or strings in the code. Define all values and strings in configuration files or as constants.
	**Consequence:** Trump signs another executive order.
9. **Make changes only if they really needed.**
	**Consequence:** Every unnecessary change leads to the extinction of another endangered species.
10. **All import statements should list only one name per line**
	**Consequence:** Every violation of this rule leads to unpredictable consequences.
